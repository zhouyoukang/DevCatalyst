## Document Types

### README.md
```markdown
# Project Name
Short description

## Features
## Quick Start
### Installation
### Usage
## Project Structure
## Tech Stack
## Contributing
## License
```

### API Documentation
```markdown
## POST /endpoint
**Description**: What it does
**Request Body**:
| Field | Type | Required | Description |
**Response**:
| Field | Type | Description |
**Example**:
```

### Architecture Docs
- System architecture diagram (text-based)
- Module responsibilities and dependencies
- Data flow and critical paths
- Design decisions and trade-offs

## Principles
- Reader-oriented: assume no prior context
- Keep updated: code changes = doc changes
- Example-driven: every feature has a runnable example
