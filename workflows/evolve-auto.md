# /review — 回顾对话、提炼经验

> 最简模式：用户说一句话，AI 干活。不搞自动化，不搞复杂架构。

## 使用方式

用户随时说以下任意一句即可触发：
- `/review`
- "回顾一下最近的对话"
- "把最新的经验整理一下"
- "看看最近有什么可以改进的"

## AI 执行步骤

### Step 1: 读取最近对话

// turbo
用 `trajectory_search` 搜索最近的对话记录，关注：
- 用户反复提到的偏好或不满
- 新发现的好用做法或模式
- 遇到的坑和解决方案
- 技术选型决策和原因

### Step 2: 提炼经验

把发现整理为简洁要点，分两类：
1. **可固化的**：反复出现 → 考虑写入规则/Memory
2. **值得记录的**：一次性但有价值 → 写入 `management/EXPERIENCE_LOG.md`

### Step 3: 展示给用户确认

把提炼结果展示给用户，格式：
```
📋 本次回顾发现 N 条经验：

**建议写入规则的：**
- [经验1] → 建议加到 [具体位置]

**建议记录的：**
- [经验2] → 写入经验日志

请确认哪些要执行。
```

### Step 4: 用户确认后执行

用户说 OK → 执行变更。用户说不要 → 跳过。就这么简单。

## 注意

- **不要自动触发**，只在用户主动说的时候执行
- **不要过度分析**，提炼 3-5 条最有价值的就够了
- **所有规则变更必须用户确认**，AI 不能自作主张
